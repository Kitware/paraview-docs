5 ## selector.js
6 
7 This script injects a version selector into the C++ and Python documentation. It selects a header element of the docs and appends a `<select>` element with a version list as well as a link to the other language documentation.
8 
9 To build with webpack:
10 * `npm install`
11 * `npm run build:release`
12 
13 The resulting script is `dist/paraview-version.js`.
14 
15 ## Shell script to automate publication
16 
17 ```
18 ./scripts/prepare-doc.sh /path/to/paraview/src /path/to/paraview/build /path/to/workdir
19 ```
20 
21 For nightly or latest (i.e. latest release) you can pass in an additional parameter on the command line
22 which is the name to use instead of `git describe` e.g.
23 
24 ```
25 ./scripts/prepare-doc.sh /path/to/paraview/src /path/to/paraview/build /path/to/workdir nightly

28 ## Updating ParaView docs
29 
30 Here are steps to update the ParaView docs manually. The instructions are have been tested
31 on a Linux system.
32 
33 * Checkout ParaView source for appropriate version.
34 * Build ParaView with `PARAVIEW_BUILD_DEVELOPER_DOCUMENTATION` CMake flag turned ON. You may also
35  want to enable all approprate features e.g. Python support, MPI support. With
36  Python, make sure the `sphinx-build:FILEPATH` points to the sphix-build script
37  for correct version of Python.
38 * Build ParaView normally. This is necessary to ensure everything is built
39  correctly.
40 * Build the `ParaViewDoc-TGZ` target e.g. `ninja ParaViewDoc-TGZ`. This will
41  generate the Doxygen and Sphinx generated docs.
42 * Now run `prepare-doc` script to push the generated docs to this repo. Provide
43  the optional `version` command line argument when generated docs for `latest`
44  or `nightly` instead of using the value obtained from `git describe` executed
45  on the source directory. The script will update and push the documentation
46  changes to the `gh-pages` branch on this repo.
47 * When adding a new version, you should edit the `versions.json` file in the
48  `gh-pages` branch to ensure it all the versions listed and labelled correctly.
49  When adding new commits to gh-pages branch, ensure that last commit is the nightly
50  doc update commit since the automatic builders ammend the last commit.
51 
52 ## How to update the generated script
53 
54 You will need to copy and commit dist/paraview-version.js from master to the root of gh-pages directory.
