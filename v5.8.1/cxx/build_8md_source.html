1 # Building ParaView
2 
3 This page describes how to build and install ParaView. It covers building for
4 development, on both Unix-type systems (Linux, HP-UX, Solaris, macOS), and
5 Windows. Note that Unix-like environments such as Cygwin and MinGW are not
6 officially supported. However, patches to fix problems with these platforms
7 will be considered for inclusion.
8 
9 ParaView depends on several open source tools and libraries such as Python, Qt,
10 CGNS, HDF5, etc. Some of these are included in the ParaView source itself
11 (e.g., HDF5), while others are expected to be present on the machine on which
12 ParaView is being built (e.g., Python, Qt).
13 
14 The first chapter is a getting started guide by OS that is very helpful if you have never
15 built ParaView before and do not know which options you need.
16 If you are looking for the generic help, please start at ## Obtaining the source
17 
18 ## Getting Started Guide
19 This is a section intended to help those that have never built ParaView before, are not
20 experienced with compilation in general or have no idea which option they may need when building ParaView.
21 If you follow this guide, you will be able to compile and run a standard version of ParaView
22 for your operating system. It will be built with the python wrapping, MPI capabilities and multithread capabilities.
23 
24  * If you are using a Linux distribution, please see [the Linux part](#linux),
25  * If you are using Microsoft Windows, please see [the Windows part](#windows),
26  * If you are using another OS, feel free to provide compilation steps.

58 #### Build
59 
60 To build ParaView developement version (usually refered as "master"), please run the following commands in a terminal :
61 ```sh
62 git clone --recursive https://gitlab.kitware.com/paraview/paraview.git
63 mkdir paraview_build
64 cd paraview_build
65 cmake -GNinja -DPARAVIEW_ENABLE_PYTHON=ON -DPARAVIEW_USE_MPI=ON -DVTK_SMP_IMPLEMENTATION=TBB -DCMAKE_BUILD_TYPE=Release ../paraview
66 ninja
67 ```
68 
69 To build a specific ParaView version, eg: v5.6.0 , please run the following commands in a terminal while replacing "tag" by the version you want to build
70 ```sh
71 git clone https://gitlab.kitware.com/paraview/paraview.git
72 mkdir paraview_build
73 cd paraview
74 git checkout tag
75 git submodule update --init --recursive
76 cd ../paraview_build
77 cmake -GNinja -DPARAVIEW_ENABLE_PYTHON=ON -DPARAVIEW_USE_MPI=ON -DVTK_SMP_IMPLEMENTATION=TBB -DCMAKE_BUILD_TYPE=Release ../paraview
78 ninja
79 ```
80 
81 #### Run
82 Double click on the paraview executable in the bin directory or run in the previous terminal
83 
84 ```sh
85 ./bin/paraview
86 ```

150 ### Building
151 
152 ParaView supports all of the common generators supported by CMake. The Ninja,
153 Makefiles, and Visual Studio generators are the most well-tested however.
154 
155 #### Prerequisites
156 
157 ParaView only requires a few packages in order to build in general, however
158 specific features may require additional packages to be provided to ParaView's
159 build configuration.
160 
161 Required:
162 
163  * [CMake][cmake]
164  - Version 3.10 or newer, however, the latest version is always recommended
165  * Supported compiler
166  - GCC 4.8 or newer
167  - Clang 4 or newer
168  - Xcode 9 or newer
169  - Visual Studio 2015 or newer
170 
171 Optional dependencies:
172 
173  * [Python][python]
174  - When using Python 2, at least 2.7 is required
175  - When using Python 3, at least 3.3 is required
176  * [Qt5][qt]
177  - Version 5.9 or newer
178 
179 ##### Installing CMake
180 
181 CMake is a tool that makes cross-platform building simple. On several systems
182 it will probably be already installed or available through system package
183 management utilities. If it is not, there are precompiled binaries available on
184 [CMake's download page][cmake-download].
185 
186 ##### Installing Qt
187 
188 ParaView uses Qt as its GUI library. Precompiled binaries are available on
189 [Qt's website][qt-download].
190 
191 Note that on Windows, the compiler used for building ParaView must match the
192 compiler version used to build Qt.
193 
194 The Linux packages for Qt 5.9 use a version of protobuf that may conflict with
195 that used by ParaView. If, when running ParaView, error messages about a
196 mismatch in protobuf versions appears, moving the `libqgtk3.so` plugin out of
197 the `plugins/platformthemes` directory has been sufficient in the past.
198 
199 #### Optional Additions
200 
201 ##### Download And Install ffmpeg (`.avi`) movie libraries
202 
203 When the ability to write `.avi` files is desired, and writing these files is
204 not supported by the OS, ParaView can use the ffmpeg library. This is generally
205 true for Linux. Source code for ffmpeg can be obtained from [the
206 website][ffmpeg].
207 
208 ##### MPI
209 
210 To run ParaView in parallel, an [MPI][mpi] implementation is required. If an
211 MPI implementation that exploits special interconnect hardware is provided on
212 your system, we suggest using it for optimal performance. Otherwise, on
213 Linux/Mac, we suggest either [OpenMPI][openmpi] or [MPICH][mpich]. On Windows,
214 [Microsoft MPI][msmpi] is required.
215 
216 ##### Python
217 
218 In order to use scripting, [Python][python] is required (versions 2.7 and 3.3).
219 Python is also required in order to build ParaViewWeb support.
220 
221 ##### OSMesa
222 
223 Off-screen Mesa can be used as a software-renderer for running ParaView on a
224 server without hardware OpenGL acceleration. This is usually available in
225 system packages on Linux. For example, the `libosmesa6-dev` package on Debian
226 and Ubuntu. However, for older machines, building a newer version of Mesa is
227 likely necessary for bug fixes and support. Its source and build instructions
228 can be found on [its website][mesa].

256 ### Building
257 
258 In order to build, CMake requires two steps, configure and build. ParaView
259 itself does not support what are known as in-source builds, so the first step
260 is to create a build directory.
261 
262 !!! note
263  On Windows, there have historically been issues with long paths to the
264  build directory. These should have been addressed, but they may appear
265  again. If any are seen, please report them to [the issue
266  tracker][paraview-issues].
267 
268 ```sh
269 mkdir -p paraview/build
270 cd paraview/build
271 ccmake ../path/to/paraview/source # -GNinja may be added to use the Ninja generator
272 ```
273 
274 CMake's GUI has input entries for the build directory and the generator
275 already. Note that on Windows, the GUI must be launched from a "Native Tools
276 Command Prompt" available with Visual Studio in the start menu.

278 #### Build Settings
279 
280 ParaView has a number of settings available for its build. These are categorized
281 as build options, capability option, feature options and miscellanoues options.
282 
283 
284 #### Build Options
285 
286 These options impact the build. These begin with the prefix `PARAVIEW_BUILD_`.
287 The common variables to modify include:
288 
289  * `PARAVIEW_BUILD_SHARED_LIBS` (default `ON`): If set, shared libraries will
290  be built. This is usually what is wanted.
291 
292 Less common, but variables which may be of interest to some:
293 
294  * `PARAVIEW_BUILD_EDITION` (default `CANONICAL`): Choose which features to
295  enable in this build. This is useful to generate ParaView builds with
296  limited features. More on this later.
297  * `PARAVIEW_BUILD_EXAMPLES` (default `OFF`): If set, ParaView's example code
298  will be added as tests to the ParaView test suite.
299  * `PARAVIEW_BUILD_DEVELOPER_DOCUMENTATION` (default `OFF`): If set, the HTML
300  documentation for ParaView's C++, Python, and proxies will be generated.
301  * `PARAVIEW_BUILD_TESTING` (default `OFF`): Whether to build tests or not.
302  Valid values are `OFF` (no testing), `WANT` (enable tests as possible), and
303  `ON` (enable all tests; may error out if features otherwise disabled are
304  required by test code).
305  * `PARAVIEW_BUILD_VTK_TESTING` (default `OFF`): Whether to build tests for the
306  VTK codebase built by ParaView. Valid values are same as
307  `PARAVIEW_BUILD_TESTING`.
308 
309 More advanced build options are:
310 
311  * `PARAVIEW_BUILD_ALL_MODULES` (default `OFF`): If set, ParaView will enable
312  all modules not disabled by other features.
313  * `PARAVIEW_BUILD_LEGACY_REMOVE` (default `OFF`): Remove legacy / deprecated
314  code.
315  * `PARAVIEW_BUILD_LEGACY_SILENT` (default `OFF`): Silence all legacy
316  / deprecated code messages.
317  * `PARAVIEW_BUILD_WITH_EXTERNAL` (default `OFF`): When set to `ON`, the build
318  will try to use external copies of all included third party libraries unless
319  explicitly overridden.
320  * `PARAVIEW_BUILD_WITH_KITS` (default `OFF`; requires CMake 3.12+): Compile
321  ParaView into a smaller set of libraries. Can be useful on platforms where
322  ParaView takes a long time to launch due to expensive disk access.
323 
324 #### Capability settings
325 
326 These settings control capabitities of the build. These begin with the prefix
327 `PARAVIEW_USE_`. The common variables to modify include:
328 
329  * `PARAVIEW_USE_QT` (default `ON`): Builds the `paraview` GUI application.
330  * `PARAVIEW_USE_MPI` (default `OFF`): Whether MPI support will be available
331  or not.
332  * `PARAVIEW_USE_PYTHON` (default `OFF`): Whether Python
333  support will be available or not.
334 
335 Less common, but potentially useful variables are:
336 
337  * `PARAVIEW_USE_VTKM` (default `ON`): Whether VTK-m based filters are enabled.
338 
339 #### Feature settings
340 
341 These settings control optional features. These begin with the prefix
342 `PARAVIEW_ENABLE_`. The common variables to modify include:
343 
344  * `PARAVIEW_ENABLE_RAYTRACING` (default `OFF`): Enable ray-tracing support
345  with OSPray and/or OptiX. Requires appropriate external libraries.
346  * `PARAVIEW_ENABLE_WEB` (default `OFF`; requires `PARAVIEW_USE_PYTHON`):
347  Whether ParaViewWeb support will be available or not.
348 
349 More advanced / less common options include:
350 
351  * `PARAVIEW_ENABLE_VISITBRIDGE` (default `OFF`): Enable support for VisIt
352  readers.
353  * `PARAVIEW_ENABLE_NVPIPE` (default `OFF`): Use [nvpipe][nvpipe] image
354  compression when communicating the GPU. Requires CUDA and an NVIDIA GPU.
355  * `PARAVIEW_ENABLE_GDAL` (default `OFF`): Enable support for reading GDAL
356  files.
357  * `PARAVIEW_ENABLE_LAS` (default `OFF`): Enable support for reading LAS
358  files.
359  * `PARAVIEW_ENABLE_OPENTURNS` (default `OFF`): Enable support for reading
360  OpenTURNS files.
361  * `PARAVIEW_ENABLE_PDAL` (default `OFF`): Enable support for reading PDAL
362  files.
363  * `PARAVIEW_ENABLE_MOTIONFX` (default `OFF`): Enable support for reading
364  MotionFX files.
365  * `PARAVIEW_ENABLE_MOMENTINVARIANTS` (default `OFF`): Enable
366  MomentInvariants filters.
367  * `PARAVIEW_ENABLE_XDMF2` (default `OFF`): Enable support for reading Xdmf2
368  files.
369  * `PARAVIEW_ENABLE_XDMF3` (default `OFF`): Enable support for reading Xdmf3
370  files.
371  * `PARAVIEW_ENABLE_FFMPEG` (default `OFF`; not available on Windows): Enable
372  FFmpeg support.
373  * `PARAVIEW_ENABLE_COSMOTOOLS` (default `OFF`; requires `PARAVIEW_USE_MPI`
374  and not available on Windows): Enable support for CosmoTools which includes
375  GenericIO readers and writers as well as some point cloud algorithms.
376 
377 
378 #### Plugin settings
379 
380 ParaView build includes several plugins. These can be enabled / disabled using the
381 following options:
382 
383  * `PARAVIEW_PLUGINS_DEFAULT` (default `ON`): Pass this flag to the command
384  line using `-DPARAVIEW_PLUGINS_DEFAULT=OFF` before the first cmake run to
385  disable all plugins by default. Note this has no impact after the first
386  cmake configure and hence must be passed on the command line itself.
387  * `PARAVIEW_PLUGIN_ENABLE_<name>` (default varies): Whether to enable a
388  plugin or not.
389  * `PARAVIEW_PLUGIN_AUTOLOAD_<name>` (default `OFF`): Whether to autoload a
390  plugin at startup or not. Note that this affects all clients linking to
391  ParaView's plugin target.
392 
393 #### Miscellaneous settings
394 ParaView uses VTK's module system to control its build. This infrastructure
395 provides a number of variables to control modules which are not otherwise
396 controlled by the other options provided.
397 
398  * `VTK_MODULE_USE_EXTERNAL_<name>` (default depends on
399  `PARAVIEW_BUILD_WITH_EXTERNAL`): Use an external source for the named third-party
400  module rather than the copy contained within the ParaView source tree.
401  * `VTK_MODULE_ENABLE_<name>` (default `DEFAULT`): Change the build settings
402  for the named module. Valid values are those for the module system's build
403  settings (see below).
404  * `VTK_GROUP_ENABLE_<name>` (default `DEFAULT`): Change the default build
405  settings for modules belonging to the named group. Valid values are those
406  for the module system's build settings (see below).
407 
408 For variables which use the module system's build settings, the valid values are as follows:
409 
410  * `YES`: Require the module to be built.
411  * `WANT`: Build the module if possible.
412  * `DEFAULT`: Use the settings by the module's groups and
413  `PARAVIEW_BUILD_ALL_MODULES`.
414  * `DONT_WANT`: Don't build the module unless required as a dependency.
415  * `NO`: Do not build the module.
416 
417 If any `YES` module requires a `NO` module, an error is raised.
418 
419 More advanced options:
420 
421  * `PARAVIEW_INITIALIZE_MPI_ON_CLIENT` (default `ON`; requires
422  `PARAVIEW_USE_MPI`): Initialize MPI on client processes by default.
423  * `PARAVIEW_USE_QTHELP` (default `ON`; requires
424  `PARAVIEW_USE_QT`): Use Qt's help infrastructure for runtime
425  documentation.
426  * `PARAVIEW_VERSIONED_INSTALL` (default `ON`): Whether to add version numbers
427  to ParaView's include and plugin directories in the install tree.
428  * `PARAVIEW_CUSTOM_LIBRARY_SUFFIX` (default depends on
429  `PARAVIEW_VERSIONED_INSTALL`): The custom suffix for libraries built by
430  ParaView. Defaults to either an empty string or `pvX.Y` where `X` and `Y`
431  are ParaView's major and minor version components, respectively.
432  * `PARAVIEW_INSTALL_DEVELOPMENT_FILES` (default `ON`): If set, ParaView will
433  install its headers, CMake API, etc. into its install tree for use.
434  * `PARAVIEW_RELOCATABLE_INSTALL` (default `ON`): If set, the install tree
435  will be relocatable to another path. If unset, the install tree may be tied
436  to the build machine with absolute paths, but finding dependencies in
437  non-standard locations may require work without passing extra information
438  when consuming ParaView.

451 ## Building editions
452 
453 A typical ParaView build includes several modules and dependencies. While these
454 are necessary for a fully functional application, there are cases (e.g. in situ
455 use-cases) where a build with limited set of features is adequate. ParaView build supports
456 this using the `PARAVIEW_BUILD_EDITION` setting. Supported values for this setting are:
457 
458 * `CORE`: Build modules necessary for core ParaView functionality.
459  This does not include rendering.
460 * `RENDERING`: Build modules necessary for supporting rendering including views
461  and representations. This includes everything in `CORE`.
462 * `CATALYST`: Build all modules necessary for in situ use cases without
463  rendering and optional components like NetCDF- and HDF5-based readers and
464  writers.
465 * `CATALYST_RENDERING`: Same as `CATALYST` but with rendering supported added.
466 * `CANONICAL` (default): Build modules necessary for standard ParaView build.
