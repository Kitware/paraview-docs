6 ###Formatting options are added for proxy documentation###
7 Documentation for proxies (and for input properties for proxies)
8 accepts reStructured text(RST) formatting options. Supported options
9 are: **bold** (use ``**bold**``), *italic* (use ``*italic*``),
10 unordered lists, and paragraphs (use an empty line). Unordered list
11 are formatted like in the following example, with an empty line
12 before and after the list.
13 
14  - first item
15  - second item
16 
17 Nested lists are not supported. Note that the text enclosed between
18 **Documentation** tags has to be aligned at column 0, (space is
19 significant in RST documents) and that we do not accept empty lines
20 between items in an unordered list. Formatted output will be displayed
21 in ParaView online help, ParaView Python documentation, and tooltips
22 displayed in ParaView client. See the **Calculator** and **Glyph**
23 filters for examples on how to format other filters.
