1 # Adding and Removing Proxies {#AddRemoveProxies}
2 
3 ## Introduction
4 
5 The proxies name and label have a major role in ParaView :
6 they are used to create graphical and python interface.
7 This has a big impact on how the users will discover features
8 but it is also important at deprecation time.
9 
10 ## Naming
11 When adding a new proxy to ParaView you should give it a name (and label).
12 You should try to find a good name at first to make it discoverable
13 and also because it is not so easy to change it afterward (see below).
14 
15 So here are some guidelines:
16 
17  1. make it concise: easier to read and remember
18  2. make it meaningful: describe what it does (that other filters do not)
19  3. avoid generic words: they can apply to too many things
20 
21 For instance:
22 `AppendLocationAttributes` was renamed to `Coordinates`
23 and `GenerateProcessIds` simply to `ProcessIds`.
24 
25 ## XML attributes
26 The `<Proxy>` and the `<Property>` tags (and derivative, such as `SourceProxy` etc)
27 have two important attributes, `name` and `label`.
28 
29 The `name` is used as an internal reference. It is the key for pvsm, the xml based statefile.
30 
31 The `label` is used for user interfaces. This includes obviously the Graphical Interface,
32 through menus, but also the python wrapping.
33 In the python case, a sanitized version of the label is used, to be a valid function name.
34 
35 If `label` is not defined, `name` is used instead.
36 
37 The label is also translatable for the Graphical Interface (not for python).
38 
39 ## Deprecation
40 When a proxy is intended for deprecation (for a future removal for instance),
41 different things should be done, specially to ensure backward compatibility.
42 
43 When a proxy is deprecated, the following files should probably be updated:
44  * `*.xml`: the servermanager file declaring the proxy. Add a `<Deprecated>` tag with explanation.
45  * `ParaViewFilters.xml`: to expose new proxy in ParaView.
46  * `vtkSMStateVersionController.cxx` for pvsm backward compatibility. It uses proxy **name**.
47  * `_backwardscompatibilityhelper.py` for python backward compatibility. It uses proxy **label**.
48  * `Documentation/release/dev/<my-change>.md` to document the change in release note
49  * `Testing/XML/*.xml`: the XML test(s) using this proxy
50  * `Testing/Python/*.py`: python tests using this proxy
51 
52 ### Python
53 To enable backwards compatibility in a python script, use the following lines
54 at the top of the script:
55 ```
56 import paraview
57 paraview.compatibility.major = 5
58 paraview.comptatbility.minor = 6
59 ```
