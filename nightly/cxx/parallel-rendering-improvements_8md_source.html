1 # Improvements to parallel rendering support
2 
3 This release includes changes to the parallel rendering code to simplify the
4 logic for developers and maintainers along with addressing some long standing
5 issues with tile displays and CAVE rendering especially when dealing with
6 split views.
7 
8 Developers developing new view types should note that the view API has been
9 simplified considerably. Earlier views needed to handle the case where all views
10 could be sharing the same render window on server processes. That is no longer
11 the case. Each view now gets a complete render window and has control over the
12 renderers added to that window. Server processes still have a share rendering
13 window which is used to display the combined results from multiple views to the
14 user via vtkPVProcessWindow, but vtkPVView subclasses don't have to handle it.
15 `vtkViewLayout` manages all the collecting of rendering results from each view
16 and posting them to a user viewable window, if and when needed.
17 
18 In CAVE mode, we now handle split views more consistently by ensuring only one
19 of the views is presented in the CAVE. Currently, it's the most recently
20 rendered view.
21 
22 Additionally, more logging entries were added that are logged under the
23 `PARAVIEW_LOG_RENDERING_VERBOSITY` category that should diagnose parallel
24 rendering issues.
