1 # Proxy XML for readers now supports multiple hints for file types
2 
3 The <Hints> element in a reader proxy can now contain multiple entries to allow
4 a reader to suport differnt filetypes with differnt descriptions.
5 
6 An example of this use case is in the ADIOS2CoreImageReader:
7 ```xml
8 <Hints>
9  <ReaderFactory extensions="bp"
10  file_description="ADIOS2 BP3 File (CoreImage)" />
11  <ReaderFactory filename_patterns="md.idx"
12  file_description="ADIOS2 BP4 Metadata File (CoreImage)" />
13  <ReaderFactory extensions="bp"
14  is_directory="1"
15  file_description="ADIOS2 BP4 Directory (CoreImage)" />
16 </Hints>
17 ```
18 THis allows a single reader to present multiple entries in the "File -> Open"
19 dialog box rather than a single combined entry.
