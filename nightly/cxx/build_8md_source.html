1 # Building ParaView
2 
3 This page describes how to build and install ParaView. It covers building for
4 development, on both Unix-type systems (Linux, HP-UX, Solaris, macOS), and
5 Windows. Note that Unix-like environments such as Cygwin and MinGW are not
6 officially supported. However, patches to fix problems with these platforms
7 will be considered for inclusion.
8 
9 ParaView depends on several open source tools and libraries such as Python, Qt,
10 CGNS, HDF5, etc. Some of these are included in the ParaView source itself
11 (e.g., HDF5), while others are expected to be present on the machine on which
12 ParaView is being built (e.g., Python, Qt).
13 
14 ## Obtaining the source
15 
16 To obtain ParaView's sources locally, clone this repository using
17 [Git][git].
18 
19 ```sh
20 git clone --recursive https://gitlab.kitware.com/paraview/paraview.git
21 ```
22 
23 ## Building
24 
25 ParaView supports all of the common generators supported by CMake. The Ninja,
26 Makefiles, and Visual Studio generators are the most well-tested however.
27 
28 ### Prerequisites
29 
30 ParaView only requires a few packages in order to build in general, however
31 specific features may require additional packages to be provided to ParaView's
32 build configuration.
33 
34 Required:
35 
36  * [CMake][cmake]
37  - Version 3.10 or newer, however, the latest version is always recommended
38  * Supported compiler
39  - GCC 4.8 or newer
40  - Clang 4 or newer
41  - Xcode 9 or newer
42  - Visual Studio 2015 or newer
43 
44 Optional dependencies:
45 
46  * [Python][python]
47  - When using Python 2, at least 2.7 is required
48  - When using Python 3, at least 3.3 is required
49  * [Qt5][qt]
50  - Version 5.9 or newer
51 
52 #### Installing CMake
53 
54 CMake is a tool that makes cross-platform building simple. On several systems
55 it will probably be already installed or available through system package
56 management utilities. If it is not, there are precompiled binaries available on
57 [CMake's download page][cmake-download].
58 
59 #### Installing Qt
60 
61 ParaView uses Qt as its GUI library. Precompiled binaries are available on
62 [Qt's website][qt-download].
63 
64 Note that on Windows, the compiler used for building ParaView must match the
65 compiler version used to build Qt.
66 
67 The Linux packages for Qt 5.9 use a version of protobuf that may conflict with
68 that used by ParaView. If, when running ParaView, error messages about a
69 mismatch in protobuf versions appears, moving the `libqgtk3.so` plugin out of
70 the `plugins/platformthemes` directory has been sufficient in the past.
71 
72 ### Optional Additions
73 
74 #### Download And Install ffmpeg (`.avi`) movie libraries
75 
76 When the ability to write `.avi` files is desired, and writing these files is
77 not supported by the OS, ParaView can use the ffmpeg library. This is generally
78 true for Linux. Source code for ffmpeg can be obtained from [the
79 website][ffmpeg].
80 
81 #### MPI
82 
83 To run ParaView in parallel, an [MPI][mpi] implementation is required. If an
84 MPI implementation that exploits special interconnect hardware is provided on
85 your system, we suggest using it for optimal performance. Otherwise, on
86 Linux/Mac, we suggest either [OpenMPI][openmpi] or [MPICH][mpich]. On Windows,
87 [Microsoft MPI][msmpi] is required.
88 
89 #### Python
90 
91 In order to use scripting, [Python][python] is required (versions 2.7 and 3.3).
92 Python is also required in order to build ParaViewWeb support.
93 
94 #### OSMesa
95 
96 Off-screen Mesa can be used as a software-renderer for running ParaView on a
97 server without hardware OpenGL acceleration. This is usually available in
98 system packages on Linux. For example, the `libosmesa6-dev` package on Debian
99 and Ubuntu. However, for older machines, building a newer version of Mesa is
100 likely necessary for bug fixes and support. Its source and build instructions
101 can be found on [its website][mesa].

103 ## Creating the Build Environment
104 
105 ### Linux (Ubuntu/Debian)
106 
107  * `sudo apt install` the following packages:
108  - `build-essential`
109  - `cmake`
110  - `mesa-common-dev`
111  - `mesa-utils`
112  - `freeglut3-dev`
113  - `ninja-build`
114  - `ninja` is a speedy replacement for `make`, highly recommended.
115 
116 *Note*: If you are using an Ubuntu-provided compiler, there is a known issue
117 with the optional Python linking. This case is hard to auto-detect, so if
118 undefined symbol errors related to Python symbols arise, setting
119 `vtk_undefined_symbols_allowed=OFF` may resolve the errors. If it does not,
120 please file a new issue.
121 
122 ### Windows
123 
124  * [Visual Studio Community Edition][visual-studio]
125  * Use "x64 Native Tools Command Prompt" for the installed Visual Studio
126  version to configure with CMake and to build with ninja.
127  * Get [ninja][ninja]. Unzip the binary and put it in `PATH`.
128 
129 ## Building
130 
131 In order to build, CMake requires two steps, configure and build. ParaView
132 itself does not support what are known as in-source builds, so the first step
133 is to create a build directory.
134 
135 !!! note
136  On Windows, there have historically been issues with long paths to the
137  build directory. These should have been addressed, but they may appear
138  again. If any are seen, please report them to [the issue
139  tracker][paraview-issues].
140 
141 ```sh
142 mkdir -p paraview/build
143 cd paraview/build
144 ccmake ../path/to/paraview/source # -GNinja may be added to use the Ninja generator
145 ```
146 
147 CMake's GUI has input entries for the build directory and the generator
148 already. Note that on Windows, the GUI must be launched from a "Native Tools
149 Command Prompt" available with Visual Studio in the start menu.

151 ### Build Settings
152 
153 ParaView has a number of settings available for its build. The common variables
154 to modify include:
155 
156  * `PARAVIEW_BUILD_SHARED_LIBS` (default `ON`): If set, shared libraries will
157  be built. This is usually what is wanted.
158  * `PARAVIEW_BUILD_QT_GUI` (default `ON`): Builds the `paraview` GUI
159  application.
160  * `PARAVIEW_USE_MPI` (default `OFF`): Whether MPI support will be available
161  or not.
162  * `PARAVIEW_USE_OSPRAY` (default `OFF`): Whether OSPRay ray-tracing support
163  will be available or not.
164  * `PARAVIEW_ENABLE_PYTHON` (default `OFF`): Whether Python
165  support will be available or not.
166  * `PARAVIEW_ENABLE_WEB` (default `OFF`; requires `PARAVIEW_ENABLE_PYTHON`):
167  Whether ParaViewWeb support will be available or not.
168  * `PARAVIEW_PLUGIN_ENABLE_<name>` (default varies): Whether to enable a
169  plugin or not.
170 
171 ParaView uses VTK's module system to control its build. This infrastructure
172 provides a number of variables to control modules which are not otherwise
173 controlled by the other options provided.
174 
175  * `VTK_MODULE_USE_EXTERNAL_<name>` (default depends on
176  `PARAVIEW_USE_EXTERNAL`): Use an external source for the named third-party
177  module rather than the copy contained within the ParaView source tree.
178  * `VTK_MODULE_ENABLE_<name>` (default `DEFAULT`): Change the build settings
179  for the named module. Valid values are those for the module system's build
180  settings (see below).
181  * `VTK_GROUP_ENABLE_<name>` (default `DEFAULT`): Change the default build
182  settings for modules belonging to the named group. Valid values are those
183  for the module system's build settings (see below).
184 
185 For variables which use the module system's build settings, the valid values are as follows:
186 
187  * `YES`: Require the module to be built.
188  * `WANT`: Build the module if possible.
189  * `DEFAULT`: Use the settings by the module's groups and
190  `PARAVIEW_BUILD_ALL_MODULES`.
191  * `DONT_WANT`: Don't build the module unless required as a dependency.
192  * `NO`: Do not build the module.
193 
194 If any `YES` module requires a `NO` module, an error is raised.
195 
196 Less common, but variables which may be of interest to some:
197 
198  * `PARAVIEW_BUILD_DEVELOPER_DOCUMENTATION` (default `OFF`): If set, the HTML
199  documentation for ParaView's C++, Python, and proxies will be generated.
200  * `PARAVIEW_ENABLE_EXAMPLES` (default `OFF`): If set, ParaView's example code
201  will be added as tests to the ParaView test suite.
202  * `PARAVIEW_ENABLE_LOGGING` (default `ON`): If set, enhanced logging will be
203  enabled.
204  * `PARAVIEW_USE_VTKM` (default `ON`): Enable VTK-m accelerated algorithms in ParaView.
205  * `PARAVIEW_ENABLE_VISITBRIDGE` (default `OFF`): Enable support for VisIt
206  readers.
207  * `PARAVIEW_BUILD_TESTING` (default `OFF`): Whether to build tests or not.
208  Valid values are `OFF` (no testing), `WANT` (enable tests as possible), and
209  `ON` (enable all tests; may error out if features otherwise disabled are
210  required by test code).
211  * `PARAVIEW_ENABLE_KITS` (default `OFF`; requires CMake 3.12+): Compile
212  ParaView into a smaller set of libraries. Can be useful on platforms where
213  ParaView takes a long time to launch due to expensive disk access.
214  * `PARAVIEW_ENABLE_CATALYST` (default `ON`): Whether to build Catalyst
215  CoProcessing support or not.

250  synchronous send commands for communication.

252  Ice-T for parallel rendering.

254  `PARAVIEW_USE_MPI`): Initialize MPI on client processes by default.
255  * `PARAVIEW_ENABLE_QT_SUPPORT` (default `OFF`; implied by
256  `PARAVIEW_BUILD_QT_GUI`): Enable Qt support.

258  `PARAVIEW_ENABLE_QT_SUPPORT`): Use Qt's help infrastructure for runtime

261  `PARAVIEW_BUILD_QT_GUI`): Build command line tools such as `pvserver` and
262  `pvrenderserver`.
263  * `PARAVIEW_USE_EXTERNAL` (default `OFF`): Whether to prefer external third
264  party libraries or the versions ParaView's source contains.
265  * `PARAVIEW_VERSIONED_INSTALL` (default `ON`): Whether to add version numbers
266  to ParaView's include and plugin directories in the install tree.
267  * `PARAVIEW_CUSTOM_LIBRARY_SUFFIX` (default depends on
268  `PARAVIEW_VERSIONED_INSTALL`): The custom suffix for libraries built by
269  ParaView. Defaults to either an empty string or `pvX.Y` where `X` and `Y`
270  are ParaView's major and minor version components, respectively.
271  * `PARAVIEW_PLUGINS_DEFAULT` (default `ON`): The state for ParaView's set of
272  enabled-by-default plugins. Note that this variable only really has an
273  effect on initial configures or newly added plugins.
