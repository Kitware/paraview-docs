1 Building ParaView
2 =================
3 
4 This page describes how to build and install ParaView. It covers building for development, on both Unix-type systems (Linux, HP-UX, Solaris, Mac), and Windows.
5 
6 ParaView depends on several open source tools and libraries such as Python, Qt, CGNS, HDF5, etc. Some of these are included in the ParaView source itself (e.g. HDF5), while others are expected to be present on the machine on which ParaView is being built (e.g. Python, Qt, CGNS).
7 
8 Adapted from the [Paraview wiki](http://www.paraview.org/Wiki/ParaView:Build_And_Install) which has more complete but dated instructions.
9 
10 Prerequisites
11 =============
12 * The ParaView build process requires [CMake](http://www.cmake.org/) version 3.3 or higher and a working compiler. On Unix-like operating systems, it also requires Make, while on Windows it requires Visual Studio and Ninja build.
13 * Building ParaView's user interface requires [Qt](http://www.qt.io/download-open-source/), version 5.6+ (5.9.\* is recommended, 5.7.\* and 5.8\* also work, 5.10 is not yet recommended). To compile ParaView, either the LGPL or commercial versions of Qt may be used. Also note that on Windows you need to choose a Visual Studio version to match binaries available for your Qt version.
14 * For Windows builds, unix-like environments such as Cygwin, or MinGW are not supported.
15 
16 Download And Install CMake
17 --------------------------
18 CMake is a tool that makes cross-platform building simple. On several systems it will probably be already installed. If it is not, please use the following instructions to install it.
19 
20 There are several precompiled binaries available at the [CMake download page](https://cmake.org/download/).
21 
22 Add CMake to your PATH environment variable if you downloaded an archive and not an installer.
23 
24 Download And Install Qt
25 --------------------------
26 ParaView uses Qt as its GUI library. Qt is required whenever the ParaView client is built with a GUI.
27 
28 * [Download a release](http://download.qt.io/official_releases/qt/).
29  - For binaries, use the latest stable version of qt-PLATFORM-opensource-VERSION.[tar.gz or zip or dmg or exe]. When downloading binaries, ensure that your compiler version matches the Qt compiler indicated. Version 5.6+ supports Visual Studio 2015.
30  - For source code, use the latest stable version of qt-everywhere-opensource-src-VERSION.[tar.gz or zip or dmg].

32 Compiler and Build Tool
33 -----------------------
34 Linux Ubuntu/Debian (16.04):
35 
36 * sudo apt install
37  - build-essential cmake git python-dev mesa-common-dev mesa-utils freeglut3-dev libhdf5-serial-dev autoconf libtool bison flex
38 * sudo python get-pip.py
39  - sudo -H pip install numpy
40  - sudo -H pip install Mako
41 * sudo apt-get install ninja-build
42  - ninja is a speedy replacement for make, highly recommended.
43 
44 **Note for Ubuntu 16.04**. The official Qt 5.9.1 binaries downloaded from `qt.io`
45 are linked against a different version of `libprotobuf` than what ParaView uses,
46 which causes runtime errors in ParaView. To avoid this, you can move the file
47 `libqgtk3.so` out from `Qt5.9.1/plugins/platformthemes`. This platform theme is
48 linked against a different `libprotobuf` than ParaView. Moving it causes it not
49 to be loaded, thereby avoiding the runtime errors with no negative effects on
50 running ParaView.

58 Optional Additions

61 ### Download And Install ffmpeg (.avi) movie libraries
62 
63 When the ability to write .avi files is desired, and writing these files is not supported by the OS, ParaView can attach to an ffmpeg library. This is generally true for Linux. Ffmpeg library source code is found here: [6](http://www.ffmpeg.org/)
64 
65 ### MPI
66 To run ParaView in parallel, an [MPI](http://www-unix.mcs.anl.gov/mpi/) implementation is required. If an MPI implementation that exploits special interconnect hardware is provided on your system, we suggest using it for optimal performance. Otherwise, on Linux/Mac, we suggest either [OpenMPI](http://www.open-mpi.org/) or [MPICH](http://www.mpich.org/). On Windows, we suggest [Microsoft MPI](https://msdn.microsoft.com/en-us/library/bb524831.aspx).
67 
68 ### Python
69 In order to use scripting, [Python](http://www.python.org/) is required (version 2.7 is supported, whereas version 3.5 support is under development).
70 Python is also required for ParaViewWeb builds.
71 
72 ### OSMesa
73 Off-screen Mesa can be used as a software-renderer for running ParaView on a server without hardware OpenGL acceleration.

76 Retrieve the Source
77 -------------------
78 * [Install Git](git/README.md) -
79  Git 1.7.2 or greater is required for development
80 
81 * [Develop ParaView](git/develop.md) - Create a fork and checkout the source code.
82  - A useful directory structure is ParaView/src for the git source tree, ParaView/build for the build tree, and if needed, ParaView/install for the install directory.
83  - Please follow the instructions in [git/develop.md] for creating a fork and setting up the repository for contributing to ParaView.
84 
85 Run CMake
86 ---------
87 * `cd ParaView/build`
88 * `cmake-gui ../src` or `ccmake ../src`
89 * Generator: choose `ninja` for the ninja build system.
90 * There are several CMake setting you may consider changing:
91 
92 | Variable | Value | Description |
93 | -------- | ----- | ------------|
94 | PARAVIEW_ENABLE_PYTHON | ON | Add python scripting support |
95 | BUILD_TESTING | ON/OFF | Build tests if you are contributing to ParaView |
96 
97 
98 Build
99 -----
100 * `cd ParaView/build`
101 * `ninja`
102 
103 ParaView will be in `bin/paraview.exe`
