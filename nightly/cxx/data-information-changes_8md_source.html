1 ## Changes to vtkPVDataInformation
2 
3 vtkPVDataInformation implementation has been greatly simplified.
4 
5 vtkPVDataInformation no longer builds a complete composite data hierarchy
6 information. Thus, vtkPVCompositeDataInformation is no longer populated
7 and hence removed. This simplifies the logic in vtkPVDataInformation
8 considerably.
9 
10 vtkPVDataInformation now provides assess to vtkDataAssembly
11 representing the hierarchy for composite datasets. This can be used by
12 application to support cases where information about the composite
13 data hierarchy is needed. For vtkPartitionedDataSetCollection, which can
14 other assemblies associated with it, vtkPVDataInformation also
15 collects those.
16 
17 For composite datasets, vtkPVDataInformation now gathers information
18 about all unique leaf datatypes. This is useful for applications to
19 determine exactly what type of datasets a composite dataset is comprised
20 of.
21 
22 vtkPVTemporalDataInformation is now simply a subclass of
23 vtkPVDataInformation. This is possible since vtkPVDataInformation no
24 longer includes vtkPVCompositeDataInformation.
