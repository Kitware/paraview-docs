1 ## Add Selection Editor
2 
3 `Selection Editor` is a new panel that allows saving and combining many selections of different types using a boolean
4 expression.
5 
6 ![Selection Editor Panel Example](../img/dev/add-SelectionEditor-PanelExample.png "Selection Editor Panel Example")
7 
8 `Selection Editor` has:
9 
10 1. `Data producer` field that is set based on the source of the active selection.
11  1. If the source of the active selection changes, the Data producer will change as well, and any saved selection
12  will be deleted.
13 2. `Element type` field that is set based on the element type of the active selection.
14  1. If a selection is saved and a new active selection is made that has a different element type, the user will be
15  prompted to decide if he wants to change the element type to add the new active selection. Doing so will result
16  in deleting the saved selection.
17 3. `Expression` string that is set by the user, and it is also automatically filled while adding new selections.
18 4. `Saved Selections` table that lists the name, which is used to define the expression, and the type of the selection.
19  1. If the user selects a selection from the `Saved Selections` table, and the active view is a render-view, the
20  selected selection will be interactively shown. Deselecting the selected selection hides the selected selection
21  from the render-view.
22 5. `Add Active Selection` button that adds the active selection to the list of saved selections.
23 6. `Remove Selected Selection` button that removes the selected saved selection from the list of saved selections.
24 7. `Remove All Selections` button that removes all saved selections from the list of saved selections.
25 8. `Activate Combined Selections` button that sets the combined saved selections as the active selection.

31 To accommodate the creation of the `Selection Editor` panel, the following changes were also completed:
32 
33 1. `SelectionQuerySource` now is implemented using `vtkSelectionSource`. `vtkQuerySelectionSource` has been removed
34  since it's no longer needed.
35 2. `AppendSelections` filter proxy has been created using `vtkAppendSelection` to combine many selections of different
36  types using a boolean expression.
37 3. Every view (render/context/spreadsheet) and the FindDataWidget now generate a `AppendSelections` proxy. Due to this
38  change, the render view can now combine all selections types including frustum-based selections. `Show frustum`
39  button has been removed, because 1) it's not used, 2) and visualizing more than one frustum would be highly confusing
40  for the user.
41 4. `vtkSelectionConverter` has been deleted because it's no longer used.
