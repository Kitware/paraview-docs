1 ## Dynamic Initialization in Plugins
2 
3 Some plugins need to invoke a function when they are loaded
4 rather than relying on static initialization.
5 Examples of this include:
6 
7 + calling third-party library initialization methods;
8 + registering functionality in one plugin that will be
9  used by functions in a separate plugin.
10 
11 This can be accomplished via the `pqAutoStartInterface` in
12 Qt-based applications or via a python initializer when building
13 ParaView with Python; however, there was not a way to do this
14 robustly in all build configurations.
15 
16 Now, the `add_paraview_plugin` macro accepts two new parameters:
17 
18 + `INITIALIZERS`, which accepts a list of free functions to invoke
19  when the plugin is loaded; and
20 + `EXTRA_INCLUDES`, which accepts a list of header files to include
21  in the plugin's implementation. (This is useful for ensuring the
22  free functions are declared before they are invoked.)
