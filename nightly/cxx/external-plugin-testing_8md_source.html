1 # Adding support for testing in external plugins
2 
3 ParaView now support XML and Python testing within external plugins.
4 
5 When adding a XML testing to an external plugin, it is now possible
6 to specify a variable, `testName_USES_DIRECT_DATA`, that will be picked up by paraview
7 testing macro. With the variable defined, no data expansion will be performed
8 which means that baseline can now be used without requiring to rely
9 on an ExternalData mechanism and directly point to baseline image
10 or data in the plugin subdirectory.
11 
12 ```
13 set (TestName_USES_DIRECT_DATA ON)
14 paraview_add_client_tests(
15  LOAD_PLUGIN PluginName
16  PLUGIN_PATH $<TARGET_FILE_DIR:PluginName>
17  BASELINE_DIR ${CMAKE_CURRENT_SOURCE_DIR}/Data/Baseline
18  TEST_SCRIPTS TestName.xml)
19 
20 ```
21 
22 When adding a python testing to an external plugin, it is now possible
23 to specify a parameter to the paraview_add_test_python function, `DIRECT_DATA`,
24 that will be picked up by VTK testing macro. With this parameter defined,
25 no data expansion will be performed which means that baseline can now be used
26 without requiring to rely on an ExternalData mechanism and directly point to baseline image
27 or data in the plugin subdirectory.
28 In order to load the plugin in the python script, it may be required to use
29 other mechanisms, see in the elevation filter example.
30 
31 ```
32 # Uses DIRECT_DATA to inform VTK to look into Data/Baseline/ for the baselines
33 paraview_add_test_python(
34  NO_RT DIRECT_DATA
35  TestPython.py)
36 ```
37 
38 See Examples/Plugins/ElevationFilter for a complete example
