1 # New CMake Module System
2 
3 VTK's module system has been updated and ParaView now works with it. CMake
4 variables have changed names:
5 
6  - `Module_X` becomes `VTK_MODULE_ENABLE_M` where X is the "library name"
7  (e.g., `vtkPVCore` and `M` is the sanitized module name (e.g.,
8  `ParaView::Core`'s sanitized name is `ParaView_Core`).
9  - `PARAVIEW_BUILD_PLUGIN_P` becomes `PARAVIEW_PLUGIN_ENABLE_P`. Enabling a
10  plugin now requests that its required modules are built.
11  - ParaView's flags forcefully disable some modules. For example, ParaView
12  can no longer be built with `PARAVIEW_USE_MPI` and still build some
13  MPI-releated modules.
14 
15 ## Server Manager XML
16 
17 Server Manager XML files may now be attached to modules using
18 `paraview_server_manager_add_xmls`:
19 
20 ```cmake
21 paraview_server_manager_add_xmls(
22  XMLS mysm.xml)
23 ```
24 
25 for the current module or:
26 
27 ```cmake
28 paraview_server_manager_add_xmls(
29  MODULE Some::VTKModule
30  XMLS mysm.xml)
31 ```
32 
33 to attach it to an external module.
34 
35 Server Manager XML contents are then provided via the
36 `paraview_server_manager_process` function:
37 
38 ```cmake
39 paraview_server_manager_process(
40  MODULES module1 module2
41  TARGET a_target_name
42  XML_FILES list_of_xml_files_variable)
43 ```
44 
45 which provides a target which may be linked to in order to access the XML files
46 attached to the given modules. See `CMake/ParaViewServerManager.cmake` for
47 details.

49 ## Plugins
50 
51 ParaView's plugin CMake API has also been improved. Documentation is in
52 `CMake/ParaViewPlugin.cmake`. The biggest change is that for a plugin to
53 provide its own classes which are used from Server Manager XML files, the
54 classes must be part of a VTK module. ParaView's plugins have been updated to
55 use this new pattern, so looking in `Plugins` for examples is recommended.
56 
57 In shared builds, plugins are now built using `add_library(MODULE)` which
58 means that they may not be directly linked to. When building clients, there
59 are options to specify plugins to load on startup.
60 
61 In static builds, the `paraview_plugin_build`'s `TARGET` parameter creates a
62 target which may be linked to in order to initialize the built plugins. See
63 its documentation for details.
64 
65 ## Clients
66 
67 The CMake API has been updated to be use CMake targets and stricter argument
68 parsing. See `CMake/ParaViewClient.cmake` for documentation.
