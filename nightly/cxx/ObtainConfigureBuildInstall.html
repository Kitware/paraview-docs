Obtain, Configure, Build, and Install Verdict

Obtaining the Verdict Source Code

The verdict source code repository is now maintained by Kitware. You can check out the source with

cvs -d :pserver:anonymous@public.kitware.com:/cvsroot/Verdict login
cvs -d :pserver:anonymous@public.kitware.com:/cvsroot/Verdict -z3 co Verdict

When asked for a password, enter "verdict".

Configuring Verdict

Verdict uses CMake to create project files. To build Verdict from source, you will need to obtain CMake.

We strongly suggest that you build Verdict in a separate directory tree from the source code. Life is just too short to explain why; if you're curious, search the CMake mailing list. To make the instructions clear, we'll assume that you have the Verdict source code in /tmp/Verdict. Configuring Verdict is then as simple as

mkdir /tmp/VerdictBuild
cd /tmp/VerdictBuild
cmake ../Verdict

At this point, you may wish to edit /tmp/VerdictBuild/CMakeCache.txt to change settings from their default values.

If you are going to include Verdict inside another library, you might want to read how to use Verdict as a Subproject.

Building and Installing Verdict

Assuming you are using CMake's Makefile generator, building and installing Verdict is accomplished with

make
make install

By default, Verdict will be installed in /usr/local. If you wish to change the location, edit your CMakeCache.txt file so that CMAKE_INSTALL_PREFIX contains a different directory and re-run cmake.
