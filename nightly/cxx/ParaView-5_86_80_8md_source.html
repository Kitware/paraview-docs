1 ParaView 5.6.0 Release Notes
2 ============================
3 
4 Major changes made since ParaView 5.5.2 are listed in this document. The full list of issues addressed by this release is available
5 [here](https://gitlab.kitware.com/paraview/paraview/-/milestones/8).

7 # General Notes
8 
9 ## State file compatibility with ParaView 4.2
10 
11 State files from ParaView 4.1 or earlier are no longer supported. To update state files from ParaView 4.1 or earlier, please use ParaView 5.5, open the state file, then save a new copy of the state file. The saved version will be readable by ParaView 5.6.

19 # New Features
20 
21 ## Added a "Reset Display Empty View" setting
22 
23 Available under the **General** settings tab, the new **Reset Display Empty View** setting
24 controls whether to reset the camera in a render view when showing a representation in a
25 previously empty view.
26 
27 Previously, ParaView always reset the camera in the view. The default setting value keeps
28 it this way, but it is now possible to disable that behavior to preserve the camera view.
29 
30 ## Annotations automatically initialized when categorical coloring is enabled
31 
32 ParaView will now attempt to automatically initialize colors and annotations when the
33 **Interpret Values as Categories** setting is enabled for a color map. It will determine the
34 categories from the currently selected source in the **Pipeline Browser**. If unsuccessful,
35 ParaView will report an error.
36 
37 ## Connection server timeout
38 
39 A control for setting the timeout to use when connecting to a server has been added
40 to the **Choose Server Configuration** dialog. In Python, a timeout parameter has been added
41 as an optional parameter to the different `Connect` methods.

94 ## Moment invariants
95 
96 Added moment invariants filters from the VTK remote module. These filters are
97 available only if the CMake option `Module_vtkFiltersMomentInvariants` is enabled
98 when configuring ParaView. The filters available are **Compute Moments**,
99 **Parallel Compute Moments**, and **Moment Invariants**.
100 
101 The theory and the algorithm are described in [Roxana Bujack and
102 Hans Hagen: "Moment Invariants for Multi-Dimensional Data"](http://www.informatik.uni-leipzig.de/~bujack/2017TensorDagstuhl.pdf)
103 
104 Developed by Roxana Bujack and Karen Tsai at Los Alamos National Laboratory.

116 ## Bounding Ruler can now measure axes along oriented bounding boxes
117 
118 In addition to measuring the X, Y, or Z axis of the axis-aligned bounding box
119 of a data set, the **Bounding Ruler** filter can now measure the Major, Medium,
120 and Minor axes of the data set's oriented bounding box. These axes are new options
121 in the **Axis** property of the **Bounding Ruler** filter. As an example, one
122 can use ParaView's selection capabilities to extract two points of interest and
123 then apply the **Bounding Ruler** with the **Axis** property set to
124 **Oriented Bounding Box Major Axis** to measure how the distance between these
125 points changes over time. [Read more...](https://blog.kitware.com/measuring-oriented-bounding-box-axes-paraview/)
126 
127 One caveat to these options is that all the points in a distributed data set need to
128 be copied to a single rank to compute the oriented bounding box. When used in a parallel
129 server setting, this may be slow or lead to memory exhaustion if the data set does
130 not fit onto one rank.

136 ## MooseXfemClip Plugin
137 
138 The **MooseXfemClip** filter has been added as a plugin to aid in the
139 visualization of results produced using the XFEM module in the
140 [MOOSE framework](http://www.mooseframework.org) developed at Idaho
141 National Laboratory.
142 
143 The MOOSE XFEM implementation uses a phantom node approach to represent
144 discontinuities in finite element solutions, which results in overlapping
145 finite elements. The **MooseXfemClip** filter clips off the non-physical
146 portions of the overlapping elements, allowing correct visualization of the discontinuous fields produced by this method.

228

394 ## Improve support of dependent components for volume representation
395 
396 When volume representation is selected and **Map Scalars** and selected data
397 array has 2 or 4 components, a new property called **Multiple Components Mapping**
398 is available.
399 
400 When this new feature is enabled:
401 * For a two-components array, the first component is mapped to the color, the
402  second component is mapped to the opacity.
403 * For a four-components array, the first, second and third components are used
404  as RGB values and the fourth component is mapped to the opacity.
405 
406 Note that:
407 * this feature is only available when **MapScalars** is *ON* ;
408 * this feature forces **Use Separate Color Map** ;
409 * it scales the color and the opacity range with the correct component.
410 
411 ## Slice representation lighting is turned off
412 
413 Lighting on the **Slice** representation had the undesirable effect of
414 darkening the slice with the default light, which is off center with
415 respect to the view and focal points. We assume the common use case
416 for this representation is to view the data as directly as possible to
417 match colors on the slice with colors in the color legend. To better
418 support this use case, lighting is disabled for this representation.
419

421 
422 It is now possible to show min and max lines in **Quartile Chart View**
423 in addition to the average and median curves when plotting summaries
424 for a dataset or extracted selection over time.

426 ## Add a Shader Replacements property to the Geometry representation
427 
428 This new property allows customization of the OpenGL shaders used for rendering
429 by providing some replacement strings for the shader produced by VTK.
430 The replacements are prescribed through a JSON string like this:
431 
432 ```json
433 [
434  {
435  "type": "fragment",
436  "original": "//VTK::Light::Impl",
437  "replacement": "gl_FragData[0]=vec4(1.0,0.0,0.0,1.0);\n"
438  },
439  // other replacements can follow
440 ]
441 ```
442 
443 In the user interface, the property is represented with a custom widget that enables
444 loading JSON from a file and saving its path as a preset. With this widget it is
445 possible to select a preset replacements file, remove a preset from the list, and edit
446 the content of the JSON string.

505
