1 String Formatting Arguments {#StringFormattingArguments}
2 =====================
3 
4 This page documents string formatting arguments that affect ParaView behavior at
5 runtime. These arguments can be used to format formattable strings.
6 
7 The class that is responsible for handling the pushing/popping of scopes (set of arguments) is `vtkPVStringFormatter`.
8 `vtkPVStringFormatter` uses the `{fmt}` library to format strings that utilize named arguments.
9 
10 The following arguments are part of the `initial argument scope`. This scope is
11 pushed and popped inside `vtkInitializerHelper`, therefore the values of those
12 arguments do NOT change at runtime. The arguments of this scope can be used in all formattable strings.
13

18 `{ENV_hostname}` | std::string | The computer's hostname. Extracted from vtksys::SystemInformation.
19 `{hostname}` | std::string | The computer's hostname. Extracted from vtksys::SystemInformation.
20 `{ENV_os}` | std::string | The computer's operating system. Extracted from vtksys::SystemInformation.

97 All the aforementioned arguments can be used in appropriate ParaView use cases to format formattable strings.
98 If a user wants to see which are the available arguments that he can use, he can try to type a wrong argument in the
99 formattable field, e.g. `{test123}`. In output messages of ParaView, the user will see a message like below:
100 
101 ```

104 Invalid format specified '{test123}'
