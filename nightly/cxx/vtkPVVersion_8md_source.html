1 ## vtkPVVersion
2 
3 * Most of the version macros have been relocated to `vtkPVVersionQuick.h`. The
4  new `PARAVIEW_VERSION_EPOCH` macro is defined as the actual
5  `PARAVIEW_VERSION_PATCH` value for release-track development and a "high"
6  value for future ParaView release development (i.e., the next minor version
7  bump). The actual patch number, and `PARAVIEW_VERSION_FULL` values are
8  available in `vtkPVVersion.h`. A new `PARAVIEW_VERSION_NUMBER_QUICK` macro
9  provides the ParaView version as a comparable value with
10  `PARAVIEW_VERSION_CHECK` using the `PARAVIEW_VERSION_EPOCH` instead of
11  `PARAVIEW_VERSION_PATCH`. The intent is to reduce the rebuild frequency
12  needed for the nightly build version bump for the in-development source.
