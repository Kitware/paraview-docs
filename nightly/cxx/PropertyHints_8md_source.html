1 Property Hints {#PropertyHints}
2 ==============
3 
4 This page documents *Property Hints*, which are xml tags accepted under *Hints*
5 for a *Property* element in Server-Manager configuration XMLs.

9 NoDefault
10 ----------
11 Skip setting run-time defaults during initialization.
12 
13 Any property with a domain that is dynamic i.e. depends on run-time
14 values such as data information or the information-only properties, often
15 resets the property value during initialization of the newly created proxy
16 (vtkSMParaViewPipelineController::PostInitializeProxy()). This, however, may
17 be undesirable and one may want the property to simply use the XML default.
18 In that case, use this property hint as follows.
19 
20  <IntVectorProperty ...>
21  ...
22  <Hints>
23  <NoDefault />
24  </Hints>
25  </IntVectorProperty>

34

37  <Hints>

39  <ComponentLabel component="0" label="X:"/>

41  </ShowComponentLabels>
42  </Hints>
43  </DoubleVectorProperty>

59 PropertyLink
60 ------------
61 Link properties to existing proxies at initialization time.
62 
63 PropertyLink hints is typically used to link a non-UI property to a
64 property on some other proxy known to exist when this proxy is created.
65 Consider this, we have a (settings, RenderViewSettings) proxy that created
66 when a new session is initialized in vtkSMParaViewPipelineController::InitializeSession().
67 Now, when a new (views, RenderView) proxy is created, we'd like a few properties
68 from the RenderView proxy to be linked to session wide instance of RenderViewSettings proxy.
69 That way, when the properties on RenderViewSettings proxy are changed, those on all
70 the RenderView proxy instances also change. We use `<PropertyLink/>` hint for such a case.
71 For example, consider the following:
72 
73  <IntVectorProperty command="SetUseOffscreenRenderingForScreenshots"
74  name="UseOffscreenRenderingForScreenshots"
75  number_of_elements="1"
76  default_values="0"
77  panel_visibility="never">
78  <Hints>
79  <PropertyLink group="settings"
80  proxy="RenderViewSettings"
81  property="UseOffscreenRenderingForScreenshots"/>
82  </Hints>
83  </IntVectorProperty>
84 
85 Here, we want the UseOffscreenRenderingForScreenshots property linked with the corresponding
86 property on the RenderViewSettings proxy. When this tag is encountered by
87 vtkSMParaViewPipelineController::PostInitializeProxy(), it will try to locate an
88 existing proxy of registered as (settings, RenderViewSettings).
89 vtkSMParaViewPipelineController::InitializeSession() ensures that all proxies encountered in
90 \c settings group are created and registered using the same group. Thus, when the
91 vtkSMParaViewPipelineController::PostInitializeProxy() looks for (settings, RenderViewSettings)
92 proxy, it will find one and then be able to setup a property link to ensure that the property
93 on the RenderView proxy is kept in sync with the property on the RenderViewSettings proxy.
94 
95 The optional attribute `unlink_if_modified` can be set to 1 if the link should
96 be broken if the user explicitly modifies the property. This is useful when
97 linking color-related properties with the active color palette, for example:
98 
99  <DoubleVectorProperty command="SetBackground"
100  default_values="0.329 0.349 0.427"
101  name="Background"
102  panel_widget="color_selector_with_palette"
103  number_of_elements="3">
104  <Hints>
105  <PropertyLink group="settings" proxy="ColorPalette" property="BackgroundColor" unlink_if_modified="1" />
106  </Hints>
107  </DoubleVectorProperty>

110 SelectionInput
111 --------------
112 Mark a input as a one that accepts a vtkSelection.
113 
114 Sometimes an input to an algorithm is a vtkSelection, not any other dataset.
115 In ParaView, vtkSelection inputs are treated specially to allow the user
116 to copy active selection from the application. Also, it keeps the application
117 from prompting the user to pick the second input when the filter is created.
118 To indicate to the UI that this input is a selection input, use the
119 `<SelectionInput/>` hint.
120 
121  <InputProperty ...>
122  ...
123  <Hints>
124  <SelectionInput />
125  </Hints>
126  </InputProperty>

128 WidgetHeight
129 ------------
130 Specify height in rows for tabular/tree widgets.
131 
132 Certain widgets that show a tablular widget of rows and columns i.e. any property widget
133 that uses a `pqTreeWidget` including the ones for `ArrayListDomain`, `ArraySelectionDomain`,
134 `EnumerationDomain`, `CompositeTreeDomain`, respect this hint to setup the default size for the
135 tabular/tree widget.
136 
137  <IntVectorProperty command="..." name="...">
138  <CompositeTreeDomain mode="all" name="tree">
139  <RequiredProperties>
140  <Property function="Input" name="Input" />
141  </RequiredProperties>
142  </CompositeTreeDomain>
143  <Hints>
144  <!-- This tag sets the height of the CompositeTreeDomain -->
145  <WidgetHeight number_of_rows="20" />
146  </Hints>
147  </IntVectorProperty>

149 Expansion
150 ---------
151 Expands all expandable items to the given depth in a tree widget.
152 
153 Certain widgets that show a tree widget -ie. any property widget
154 that uses a `pqTreeWidget` including the ones for `ArrayListDomain`, `ArraySelectionDomain`,
155 `EnumerationDomain`, `CompositeTreeDomain`- respect this hint to setup the default expansion depth
156 for the tree widget.
157 
158 0 is the minimal expansion, -1 is expand all.
159 
160  <IntVectorProperty command="..." name="...">
161  <CompositeTreeDomain mode="all" name="tree">
162  <RequiredProperties>
163  <Property function="Input" name="Input" />
164  </RequiredProperties>
165  </CompositeTreeDomain>
166  <Hints>
167  <!-- This tag sets the expansion depth of the CompositeTreeDomain -->
168  <Expansion depth="3" />
169  </Hints>
170  </IntVectorProperty>

172 FileChooser
173 ------------
174 Specify supported extensions to list for `pqFileChooserWidget` dialog.
175 
176 For properties using FileListDomain to show a file chooser widget on the
177 Properties panel, sometimes we want to provide a list of extensions expected. In
178 that case, one can use this hint. Note, this is not intended for specifying
179 extensions that a reader supports. For that one uses the `<ReaderFactory>` hint
180 described in [ProxyHints](@ref ProxyHints). Multiple `FileChooser` hints may be
181 specified to show multiple extensions with different description texts.
182 
183 By default, the file chooser widget will accept only existing files.
184 It is possible to add `<UseDirectoryName/>` in the `<Hints>` section to accept only directories,
185 `<AcceptAnyFile/>` to accept any filename to export data, or
186 `<BrowseLocalFileSystem/>` to browse local file-system irrespective of the
187 connection type.
188 
189  <StringVectorProperty animateable="0"
190  command="SetQFileName"
191  name="QFileName"
192  number_of_elements="1"
193  panel_visibility="default">
194  <FileListDomain name="files" />
195  <Documentation>This property specifies the .q (solution) file name for
196  the PLOT3D reader.</Documentation>
197  <Hints>
198  <FileChooser extensions="q" file_description="Solution files" />
199  </Hints>
200  </StringVectorProperty>

202 OmitFromLoadAllVariables
203 ------------
204 Specify that a dataset property be excluded when "Load All Variables" toggle is selected.
205 
206 ParaView has a global toggle named "Load All Variables" that
207 automatically selects all variables in a dataset when loading a
208 file. This hint allows certain values to be omitted from that list
209 (e.g. sidesets, edgesets) so they will not be included by default. The
210 user can still manually select these values to be loaded.
211 
212  <StringVectorProperty command="SetSideSetArrayStatus"
213  element_types="2 0"
214  information_property="SideSetInfo"
215  name="SideSetArrayStatus"
216  number_of_elements_per_command="2"
217  repeat_command="1">
218  <ArraySelectionDomain name="array_list">
219  <RequiredProperties>
220  <Property function="ArrayList"
221  name="SideSetInfo" />
222  </RequiredProperties>
223  </ArraySelectionDomain>
224  <Documentation>An Exodus II file may define subsets of all the
225  <i>boundaries</i>of all the elements in a file as sets in their own
226  right. This property specifies which of those sets should be loaded.
227  Variables, such as boundary conditions, may then be defined over these
228  sets by specifying a single number per side. For example, a hexahedron
229  has 18 sides: 6 faces and 12 edges. Any of these sides may be
230  individually called out in a set and assigned a result value. The
231  accompanying SideSetResultArrayStatus property specifies which
232  variables defined over those sets should be loaded.</Documentation>
233  <Hints>
234  <OmitFromLoadAllVariables />
235  </Hints>
236  </StringVectorProperty>

238 ProxySelectionWidget
239 ---------------------
240 Specify options to configure `pqProxySelectionWidget`, typically used for
241 proxy-properties with a proxy-list domain.
242 
243 To hide the combo-box widget, add `visibility="0"` attribute.
244 To disable the combo-box widget, so that the user cannot change the selection,
245 add `enabled="0"` attribute.
246 
247  <ProxyProperty name="Format">
248  <ProxyListDomain name="proxy_list">
249  <Group name="screenshot_writers" />
250  </ProxyListDomain>
251  <Hints>
252  <ProxySelectionWidget enabled="0" />
253  </Hints>
254  </ProxyProperty>

284 DoubleRangeSliderPropertyWidget
285 -------------------------------
286 
287 Customize the `pqDoubleRangeSliderPropertyWidget` with these hints:
288 
289 * `HideResetButton` - when an XML element with this name is present, the reset range button will not be shown
290 * `MinimumLabel` - the `text` attribute in this element will be used for the minimum slider widget label
291 * `MaximumLabel` - the `text` attribute in this element will be used for the maximum slider widget label
292 
293 The snippet below shows these hints in use.
294 
295  <DoubleVectorProperty command="SetClippingLimits"
296  default_values="0.8 1.2"
297  name="ClippingLimits"
298  panel_visibility="advanced"
299  number_of_elements="2"
300  panel_widget="double_range">
301  <DoubleRangeDomain
302  max="2.0"
303  min="0.65"
304  name="range" />
305  <Hints>
306  <MinimumLabel text="Near Clipping Limit"/>
307  <MaximumLabel text="Far Clipping Limit" />
308  <HideResetButton/>
309  </Hints>
310  </DoubleVectorProperty>
